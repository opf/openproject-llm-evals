We need to fix how the LDAP sync handles structure. Right now, it just dumps everyone into flat groups, but admins are complaining because their actual organizational hierarchy is lost. If they have nested OUs or groups in LDAP, they want to see that reflected in OpenProject.
I was thinking we should give them a toggle or a config option: do they want the sync to create standard groups like it does now, or should it map to 'Departments'? Since LDAP already enforces that a user only belongs to one OU/group, mapping it to our Department model should be pretty clean.
The big thing is the nesting. If a group is a child of another in LDAP, we need to replicate that parent-child relationship here. This is strictly an admin-level configuration—normal users won't see these settings. We’ll definitely need to translate the new UI labels for this config since it's going into the admin panel.
